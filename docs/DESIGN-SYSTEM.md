# DeskPet Agent 设计系统

> 适用于 ESP32-S3 圆形 AMOLED（466×466）+ LVGL 9。
> 本文件为 UI 设计交付物，供研发实现时引用。

## 1. 设计原则

1. **纯黑优先**：背景使用 `#0A0A0C`，最大化 AMOLED 关闭像素比例，降低功耗、延缓烧屏。
2. **Source Halo 为签名元素**：屏幕外缘细环作为设备情绪灯，统一四屏视觉。
3. **圆形优先**：所有内容落在安全圆（半径 ≈210 px）内；不用矩形窗口概念。
4. **克制用色**：除 Halo 与关键状态外，界面保持低饱和、高对比。
5. **信息分层**：同一屏只做一件事，长内容通过上下滑翻看。

## 2. 色彩系统

| Token | Hex | 用途 |
|---|---|---|
| `void` | `#0A0A0C` | 屏幕背景、纯黑底色 |
| `stone` | `#1E1F25` | 卡片、未激活 tile、分隔面 |
| `mist` | `#E8ECF2` | 主文字、图标、宠物线条 |
| `workblue` | `#58B4FC` | WorkBuddy 来源、WiFi/BLE 激活态、信息 |
| `claudepurple` | `#B18CFF` | Claude Code 来源、通知脉冲 |
| `mint` | `#5EE9B0` | 成功、下一步、传输激活、充电完成 |
| `coral` | `#F87171` | 错误/断连、录音脉冲 |
| `amber` | `#FBBF24` | 权限请求、警告 |

### 语义色使用规则

- **来源色**：任何与 Agent 来源相关的标记（通知、会话点、Halo）使用 `workblue` 或 `claudepurple`。
- **成功/动作**：「下一步」、已发送、充电完成、传输在线使用 `mint`。
- **警告/阻塞**：权限请求用 `amber`；错误、断连、录音状态用 `coral`。
- **背景层**：能用 `void` 就不用 `stone`；卡片仅在承载文字/控件时使用。

## 3. 字体与排版

### 3.1 字体策略

- **英文、数字、符号**：使用 LVGL 9 内置 `LV_FONT_MONTSERRAT_*`。
- **中文**：需自行转换 **Noto Sans CJK SC** 为 LVGL 字库，建议尺寸 12/14/16 px，字重 Regular / Medium 两种即可。整个 UI 共用一套中文字体。

### 3.2 字号层级

| Token | 尺寸 | 行高 | 用途 |
|---|---|---|---|
| `T1` | 24 px | 32 px | 大数字（电量 %、状态码）、空屏标题 |
| `T2` | 16 px | 22 px | 卡片标题、会话名、屏幕标题 |
| `T3` | 14 px | 20 px | 正文、按钮文字、tile 标签 |
| `T4` | 12 px | 16 px | 时间戳、提示语、角标、来源标签 |

### 3.3 排版规则

- 圆形屏优先单行标签，过长用省略号截断。
- 正文最多 3 行，超出通过上下滑翻看。
- 所有文本在安全圆内，距边缘 ≥24 px。

## 4. 布局与间距

### 4.1 画布

- 分辨率：466 × 466 px，圆心 (233, 233)。
- 安全圆：半径 210 px（直径 420 px）。
- 边缘安全区：距物理边缘 24 px 内不放置可点击/可读内容。

### 4.2 间距尺度

| Token | 值 | 用途 |
|---|---|---|
| `xs` | 4 px | 图标与文字间距、细缝 |
| `sm` | 8 px | 同组元素内间距 |
| `md` | 16 px | 卡片内边距、默认间隙 |
| `lg` | 24 px | 区块间距、边缘安全距离 |
| `xl` | 32 px | 大区域分隔 |

### 4.3 圆角

- 卡片：14 px（小卡）/ 16 px（大卡）。
- 标签、按钮、胶囊：全圆 pill（高度的一半）。
- 圆形按钮：48 px / 56 px 直径。
- Tile：12 px 圆角。

## 5. Source Halo（签名元素）

屏幕外缘 2–4 px 的细环，作为全局状态光。所有屏幕共用同一 Halo。

| 状态/来源 | Halo 颜色 | 动画 |
|---|---|---|
| WorkBuddy 激活 | `workblue` | 顺时针匀速扫过，周期 2 s |
| Claude Code 激活 | `claudepurple` | 逆时针 shimmer，周期 2 s |
| 系统/成功 | `mint` | 稳定微光，无旋转 |
| 权限请求 | `amber` | 慢呼吸，周期 2 s |
| 错误 / 断连 | `coral` | 快速三连闪（300 ms ×3）后常亮 |
| 新通知 | 来源色（蓝/紫） | 径向脉冲，周期 1.5 s，持续 5 s 或用户操作后停止 |

> Halo 在屏幕切换时保持颜色和动画连续，强化「同一画布」感知。

## 6. 图标与图形

- 使用 LVGL 内置符号（`LV_SYMBOL_*`）优先；不足时使用简笔 SVG 路径。
- 宠物脸用矢量路径绘制，不使用图片/GIF，节省 PSRAM。
- 图标尺寸：tile 内 20 px；按钮内 24 px；状态图标 16 px。

## 7. 动效

| 场景 | 时长 | 曲线 | 说明 |
|---|---|---|---|
| 屏幕切换 | 250 ms | ease-out | 水平平移 ±30 px，背景微视差 15 px |
| 控制中心展开/收起 | 200 ms | ease-out | 从顶部向下覆盖，带透明度 0→1 |
| 点击反馈 | 150 ms | ease-out | scale 1.0 → 0.96 → 1.0 |
| 开关切换 | 150 ms | ease-out | 颜色填充 + 指示点缩放 |
| 宠物 idle 呼吸 | 3 s | ease-in-out | scale 1.00 → 1.03，循环 |
| Source Halo 旋转 | 2 s | linear | 连续循环 |
| 通知脉冲 | 1.5 s | ease-in-out | 不透明度 0.6 → 1.0 |
| 语音录音脉冲 | 800 ms | ease-in-out | coral 环不透明度脉冲 |
| 宠物被戳抖动 | 200 ms | ease-out | 旋转 ±6°，循环 2 次 |

### 烧屏保护

- 所有循环动画在 30 s 无状态变化后自动暂停，等待下一次触发再恢复。
- 不使用纯白色大面积静态画面。
- Source Halo 位置固定，但颜色和动画持续变化，降低固定像素风险。

## 8. 屏幕通用结构

每个全屏视图共享以下层次：

1. **底层**：`void` 纯黑背景。
2. **Source Halo 层**：外缘状态光环。
3. **内容层**：按屏幕规则排布的卡片/控件。
4. **提示层**：底部 12 px 提示文字（如「右滑下一个会话」）。

## 9. 手势与交互

| 手势 | 上下文 | 行为 |
|---|---|---|
| 左滑 | 主页 | 进入负一屏 |
| 右滑 | 主页 | 进入会话页（最近会话） |
| 右滑 | 会话页 | 下一个会话（循环） |
| 左滑 | 会话页 | 上一个会话；首个会话再左滑 → 主页 |
| 下滑 | 仅主页 | 拉出控制中心 |
| 上滑 / 点手柄 | 控制中心 | 收起回主页 |
| 上滑 / 下滑 | 负一屏 / 会话页 / 展开通知 | 滚动内容 |
| 底边上滑（起点距底 ≤24 px） | 任意屏 | 回主页 |
| 点按宠物 | 主页 | 抖动反馈 |
| 长按宠物 | 主页 | 快捷菜单（v2，预留） |
| 点按语音按钮 | 负一屏 | 启动语音流程 |

## 10. 与研发的衔接

- 颜色、字号、间距均在本文件以 Token 形式定义；SPEC 文档引用 Token 名而非硬编码 HEX。
- SVG 视觉稿使用本文件配色，作为实现时的像素参考。
- 中文字体需研发根据目标尺寸生成 LVGL 字库文件（`.c`），建议文件名：`font_noto_cjk_12.c`、`font_noto_cjk_14.c`、`font_noto_cjk_16.c`。
